# 페페의모험 — 분할 구조 안내

## 파일 구성
```
index.html                     HTML 셸 (스크립트 로드 순서 정의 — 순서 변경 금지)
css/style.css                  전체 스타일
js/
├── 01-sfx.js                  효과음 합성 + 공용 유틸
├── 02-assets.js               에셋 로더 (이미지 폴백·프리로드·BGM·효과음 파일)
├── 03-data.js                 게임 데이터 (몬스터/아이템/음식/훈장/대사/칭호 조건)
├── 04-engine.js               게임 엔진 (상태·저장/로드·레벨업·칭호·호감도)
├── 05-views-core.js           뷰 공통 (사이드바·로그·다이얼로그 씬·화면 전환)
├── 06-views-menu.js           타이틀 / 캐릭터 설정 / 튜토리얼
├── 07-views-village.js        마을 / NPC 씬 / 훈장 보관함 / 지도
├── 08-views-shop.js           상점 / 판매 / 식당
├── 09-views-system.js         시스템 메뉴 / 칭호 도감 / 게임오버
├── 10-views-adventure.js      갈림길 / 탐험 / 상자 / 전투
├── 11-views-events.js         사막 이벤트 / 훈장 수여식 / 엔딩 크롤
├── 12-views-casino.js         도박장 (슬롯머신 / 블랙잭 / 텍사스 홀덤)
└── 13-boot.js                 부팅 (게임 시작점)
```

## 작업 가이드
- **밸런스 수정** → `03-data.js` 의 MONSTERS / ITEMS / FOODS 배열
- **새 화면 추가** → `js/` 에 새 파일 생성 후 `index.html` 의 13-boot.js **앞에** script 태그 추가
- **저장 데이터에 필드 추가** → `04-engine.js` 의 resetData()에 기본값을 넣으면
  기존 세이브와 자동 호환됩니다 (로드 시 기본값 위에 저장값을 덮어쓰는 구조)
- 파일들은 전역 스코프를 공유합니다. 정의(데이터·엔진)가 사용(뷰)보다
  먼저 로드되도록 번호 순서를 유지하세요.
